概述

Hermes Agent 是 Nous Research 开发的智能 AI 助手，运行在 WSL (Windows Subsystem for Linux) 环境中，可以直接访问 Windows 文件系统。它集成了丰富的工具集，支持终端执行、文件操作、GitHub 集成、科研搜索、创意生成等多种能力。

运行环境：WSL Ubuntu · Python 3.14 · 可通过 /mnt/c/ 访问 Windows 文件系统

核心特性

  • 终端执行 — 前台/后台运行命令，进程管理，PTY 交互模式
  • 文件操作 — 读写文件、精确补丁编辑、正则搜索
  • Python 执行 — 内置工具调用的 Python 脚本运行环境
  • 跨会话记忆 — 持久化记忆 + 历史会话全文检索
  • 技能系统 — 可复用的工作流模板，覆盖 20+ 领域
  • 子代理 — 并行任务委派，多 Agent 协作
  • 定时任务 — Cron 调度，自动化执行
  • GitHub 深度集成 — PR、Issue、Code Review 全流程

核心工具详解

Hermes Agent 拥有 15+ 个内置工具，覆盖从文件操作到任务编排的完整工作流。

技能使用方式

技能加载后会自动提供领域专业知识、最佳实践、具体命令和常见陷阱：

# 加载技能
skill_view(name="github-pr-workflow")

# Agent 自动遵循技能中的指令完成后续操作
# 无需手动记忆复杂的工作流步骤
提示：技能会在相关任务中自动匹配加载，你也可以明确指定要使用的技能。

最佳实践

让 Agent 更高效

  1. 明确目标 — 描述你想要的结果，而非具体步骤。Agent 会自行规划执行路径。
  2. 提供上下文 — 指明文件路径、项目位置、相关依赖，减少 Agent 的探索成本。
  3. 善用技能 — 对于已知的工作流（PR、Code Review、部署），Agent 会加载对应技能获得最佳实践。
  4. 利用子代理 — 多个独立任务可以并行委派，提高效率。
  5. 保存记忆 — 让 Agent 记住你的偏好和项目约定，跨会话保持一致性。

常见陷阱

  • 不要描述步骤而非目标 — "依次读取文件 A、B、C，然后修改第 5 行"不如"优化这三个文件的错误处理"高效
  • 不要忽略技能 — 复杂任务不加载技能，Agent 可能遗漏领域最佳实践
  • 大文件用分页读取 — 使用 read_file 的 offset 和 limit 参数避免上下文溢出
  • 注意 WSL 路径映射 — Windows 路径需转换为 /mnt/c/... 格式
