DeepSeek — 概述

DeepSeek（深度求索）是国内顶级的 AI 大模型厂商，提供 OpenAI 兼容的 API 接口。其模型在编程、推理、中文理解方面表现优异，且价格极具竞争力。

官网：https://platform.deepseek.com  |  文档：https://api-docs.deepseek.com  |  状态页：https://status.deepseek.com

核心优势

  • 性价比高 — 模型能力接近 GPT-4/Claude，价格仅为 1/10 ~ 1/50
  • 中文友好 — 中文理解和生成能力顶尖
  • 编程能力强 — 在代码生成、Debug、架构设计方面表现出色
  • 国内直连 — 无需代理，国内网络直接访问
  • 1M 上下文 — V4 系列支持 100 万 token 上下文窗口
  • OpenAI 兼容 — 无缝替换 OpenAI SDK，改 base_url 即可

DeepSeek — 获取 API Key

  1. 访问 platform.deepseek.com 注册/登录
  2. 进入 API Keys 页面
  3. 点击「创建新的 API Key」
  4. 复制 key（格式：sk-xxxxxxxx）
  5. 在 Usage 页面充值（最低 ¥10）
定价参考：deepseek-chat ¥1/百万输入 tokens，¥4/百万输出 tokens（约为 GPT-4o 的 1/10）

DeepSeek — Hermes 集成

1. 配置 API Key

编辑 ~/.hermes/.env：

# ~/.hermes/.env
DEEPSEEK_API_KEY=sk-你的DeepSeek密钥
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1

2. 配置 Provider

编辑 ~/.hermes/config.yaml：

# ~/.hermes/config.yaml
model:
  default: deepseek-chat
  provider: deepseek
  base_url: https://api.deepseek.com/v1
  api_key: ${DEEPSEEK_API_KEY}

providers:
  deepseek:
    api_key: ${DEEPSEEK_API_KEY}
    base_url: https://api.deepseek.com/v1

3. 验证连接

# 方式一：CLI 启动
hermes chat --provider deepseek --model deepseek-chat -q "回复 CONNECTED"

# 方式二：会话内切换
/model deepseek:deepseek-v4-pro

# 方式三：curl 测试
curl -s https://api.deepseek.com/v1/models \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY"
提示：DeepSeek 国内直连无压力，不需要代理。是 Agent 场景的首选方案。

Agnes AI — 概述

Agnes AI 是新加坡 Sapiens AI 公司推出的全模态 AI 平台，提供文本、图像、视频三大类模型。2026 年 6 月起核心模型 API 无限期免费开放。

官网：https://agnes-ai.com  |  管理后台：https://platform.agnes-ai.com  |  API 端点：https://apihub.agnes-ai.com/v1

核心优势

  • 免费开放 — 核心模型 2026 年 6 月起无限期免费
  • 全模态 — 文本 + 图像 + 视频，一个平台搞定
  • Agent 优化 — agnes-2.0-flash 专为 Agent 工作流设计
  • OpenAI 兼容 — 标准 Chat Completions API，无缝迁移
  • 256K 上下文 — 超长上下文窗口
⚠️ 端点区分：Agnes 的图片和视频模型使用独立 API 端点（/v1/images/generations 和 /v1/videos），不走 /v1/chat/completions。在 Hermes 中它们不能通过 /model 切换，否则返回 404。详见下方模型列表。

Agnes AI — 获取 API Key

  1. 访问 platform.agnes-ai.com 注册账号
  2. 进入 API Keys 管理页面
  3. 创建新的 API Key（格式：sk-La7L4...）
  4. 复制保存 key

Agnes AI — Hermes 集成

1. 配置 API Key

编辑 ~/.hermes/.env：

# ~/.hermes/.env
AGNES_API_KEY=sk-La7L4...你的Agnes密钥
AGNES_BASE_URL=https://apihub.agnes-ai.com/v1

2. 添加 Provider（仅限聊天模型）

编辑 ~/.hermes/config.yaml，在 providers 中添加 agnes，并限定 models 白名单——只注册聊天模型，防止图片/视频模型出现在选择器中被误选导致 404：

# ~/.hermes/config.yaml
model:
  default: agnes-2.0-flash          # 默认使用 Agnes
  provider: agnes
  base_url: https://apihub.agnes-ai.com/v1
  api_key: ${AGNES_API_KEY}

providers:
  deepseek:
    api_key: ${DEEPSEEK_API_KEY}
    base_url: https://api.deepseek.com/v1
  agnes:
    api_key: ${AGNES_API_KEY}
    base_url: https://apihub.agnes-ai.com/v1
    models:                                # 白名单：仅聊天模型
      agnes-2.0-flash:
        name: Agnes 2.0 Flash
        type: chat
      agnes-1.5-flash:
        name: Agnes 1.5 Flash
        type: chat
⚠️ 为何不把图片/视频模型也放在 models 里？因为 agnes-image-2.1-flash、agnes-image-2.0-flash、agnes-video-v2.0 不是聊天模型——它们分别走 /v1/images/generations 和 /v1/videos 端点。Hermes 把 /model 切换的模型一律当聊天模型，请求发到 /v1/chat/completions，导致 HTTP 404 NotFoundError。

3. 聊天模型切换

# 启动时指定 Agnes 聊天模型
hermes chat --provider agnes --model agnes-2.0-flash

# 会话内切换（仅聊天模型！）
/model agnes:agnes-2.0-flash
/model agnes:agnes-1.5-flash

# ❌ 不要这样！图片/视频模型走不了聊天端点，会报 404
/model agnes:agnes-image-2.1-flash    # → HTTP 404
/model agnes:agnes-video-v2.0         # → HTTP 404

4. 图片/视频生成（正确方式）

图片和视频模型只能通过正确的 API 端点调用：

文生图（agnes-image-2.1-flash）

# POST /v1/images/generations
curl -X POST https://apihub.agnes-ai.com/v1/images/generations \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $AGNES_API_KEY" \
  -d '{
    "model": "agnes-image-2.1-flash",
    "prompt": "一只橙色小猫坐在木桌上，暖光，照片级真实",
    "size": "1024x1024"
  }'

图生图（agnes-image-2.0-flash）

# POST /v1/images/generations
curl -X POST https://apihub.agnes-ai.com/v1/images/generations \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $AGNES_API_KEY" \
  -d '{
    "model": "agnes-image-2.0-flash",
    "prompt": "转换成水彩画风格",
    "image": "https://example.com/input.jpg",
    "tags": ["img2img"],
    "seed": 42
  }'

文生视频（agnes-video-v2.0）— 异步

# Step 1: 创建视频任务 → POST /v1/videos
TASK_ID=$(curl -s -X POST https://apihub.agnes-ai.com/v1/videos \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $AGNES_API_KEY" \
  -d '{
    "model": "agnes-video-v2.0",
    "prompt": "一只猫在阳光海滩上行走，电影级画质",
    "width": 1152, "height": 768,
    "num_frames": 81, "frame_rate": 24
  }' | jq -r '.task_id')

# Step 2: 轮询直到完成 → GET /v1/videos/{task_id}
curl -s https://apihub.agnes-ai.com/v1/videos/$TASK_ID \
  -H "Authorization: Bearer $AGNES_API_KEY" | jq '.status'

# num_frames 合法值（8n+1）：81, 121, 161, 201, 241, 281, 321, 361, 401, 441
# 时长 = num_frames / frame_rate
Skill 快捷方式：在 Hermes 中输入 /agnes-ai 加载 Agnes 图片/视频 skill，可直接用 Python 脚本调用以上所有端点，无需手写 curl。
⚠️ 网络问题：如果遇到 Recv failure: Connection was reset（SSL 连接重置），说明 Agnes 在你当前网络不可达。解决方案：
  • 设置代理：export HTTPS_PROXY=http://127.0.0.1:你的代理端口
  • 或者直接用 DeepSeek（国内直连无压力）

多 Provider 切换

Hermes 支持在会话中随时切换模型和 Provider，无需重启。

切换命令（仅聊天模型）

命令效果
/model deepseek:deepseek-chat切换到 DeepSeek 默认模型
/model deepseek:deepseek-v4-pro切换到 DeepSeek 旗舰模型
/model agnes:agnes-2.0-flash切换到 Agnes 主力聊天模型
/model agnes:agnes-1.5-flash切换到 Agnes 轻量聊天模型
❌ 图片/视频模型不能切换：/model agnes:agnes-image-2.1-flash 等命令会报 HTTP 404，因为图片和视频模型走的是 /v1/images/generations 和 /v1/videos 端点，不是聊天端点。详见上方「Agnes Hermes 集成」中的正确用法。

完整 config.yaml（双 Provider）

model:
  default: deepseek-chat           # 默认模型
  provider: deepseek                # 默认 Provider
  base_url: https://api.deepseek.com/v1
  api_key: ${DEEPSEEK_API_KEY}

providers:
  deepseek:
    api_key: ${DEEPSEEK_API_KEY}
    base_url: https://api.deepseek.com/v1
  agnes:
    api_key: ${AGNES_API_KEY}
    base_url: https://apihub.agnes-ai.com/v1
    models:                                # 仅聊天模型——图片/视频走独立端点
      agnes-2.0-flash:
        name: Agnes 2.0 Flash
        type: chat
      agnes-1.5-flash:
        name: Agnes 1.5 Flash
        type: chat

fallback_providers: []                # 可配置降级链路
降级策略：可在 fallback_providers 中配置 ["deepseek"]，当 Agnes 不可用时自动降级到 DeepSeek。

常见问题排查

401 认证失败

AuthenticationError [HTTP 401]: 无效的令牌
原因 & 解决：
  1. API Key 错误或过期 → 重新获取 key 更新 .env
  2. config.yaml 中硬编码了旧 key → 用 ${ENV_VAR} 引用 .env
  3. 账户余额不足 → 检查平台充值状态
  4. Key 没有对应模型的访问权限 → 确认账户已开通该模型

404 NotFoundError（Agnes 图片/视频模型）

NotFoundError [HTTP 404]: ***.NotFoundError: OpenAIException - {"detail":"Not Found"}
原因：使用了 /model agnes:agnes-image-xxx 或 /model agnes:agnes-video-xxx，这些不是聊天模型，不存在于 /v1/chat/completions 端点。
解决：
  1. 切回聊天模型：/model agnes:agnes-2.0-flash
  2. 用正确端点调用图片生成：POST /v1/images/generations（详见 Agnes Hermes 集成）
  3. 用正确端点调用视频生成：POST /v1/videos（异步轮询）
  4. 或加载 skill：/agnes-ai

SSL 连接重置（Agnes 专用）

Recv failure: Connection was reset
schannel: failed to receive handshake, SSL/TLS connection failed
原因：Agnes AI 服务器（Cloudflare IP）被网络屏蔽。
解决：挂代理，或改用 DeepSeek 直连。

Endpoint 没有切换

改完 .env 和 config.yaml 后，请求仍然发到旧地址。

排查步骤：
  1. 确认修改的是 WSL 内 的 ~/.hermes/config.yaml，不是 Windows 侧的
  2. 检查 config.yaml 中 model.base_url 是否覆盖了 provider 的 base_url
  3. 检查 model.api_key 是否硬编码了字符串而非 ${VAR} 引用
  4. 重启 Hermes（config.yaml 和 .env 在启动时加载）

辅助服务也用 Agnes Key

Vision、Web Extract、Compression 等辅助服务可能单独配置了 key。

# 检查 auxiliary 部分
grep -A2 'api_key:' ~/.hermes/config.yaml | head -20
建议统一使用 env var 引用，方便批量切换：api_key: ${DEEPSEEK_API_KEY}
